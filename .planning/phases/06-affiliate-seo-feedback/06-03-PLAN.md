<task type="auto">
  <name>Task 1: Create Refresh Candidates Writer workflow</name>
  <files>growth/06_Refresh_Candidates_Writer.json</files>
  <action>
Create new n8n workflow: name "06 — Refresh Candidates Writer" (or "Refresh Candidates Writer").

1. **Schedule Trigger:** e.g. "0 5 * * *" (daily 5 AM) so candidates are ready before 8 AM orchestrator run. Document in notes.

2. **Execute Workflow — Config Loader:** Same pattern as Viral Amplifier; workflowId reference to core/01_Config_Loader. Need: GOOGLE_SHEET_ID (or SPREADSHEET_ID), REFRESH_CANDIDATES_TAB, GA4_PROPERTY_ID, GOOGLE_ANALYTICS_TOKEN, REFRESH_VIEWS_MIN (optional; default e.g. 100 or reuse VIRAL_VIEWS_7D_MIN from config).

3. **HTTP Request — GA4 runReport:** Copy pattern from growth/HowTo-Genie v4.0 — Viral Content Amplifier Engine.json: POST https://analyticsdata.googleapis.com/v1beta/properties/{{ GA4_PROPERTY_ID }}:runReport, Authorization Bearer {{ GOOGLE_ANALYTICS_TOKEN }}, body with dateRanges (7daysAgo–today), dimensions pagePath, pageTitle, metrics screenPageViews, averageSessionDuration, engagementRate, bounceRate, orderBys screenPageViews desc, limit 50. Use expressions for config: $('⚙️ Load Config').item.json.GA4_PROPERTY_ID etc.

4. **Code — Filter by threshold and shape rows:** Parse GA4 response rows; filter where screenPageViews >= (config.REFRESH_VIEWS_MIN || 100). Map each row to: date (today ISO slice), post_url (WORDPRESS_URL + pagePath), post_title (pageTitle), views_7d, keyword (extract from title or pagePath for Agent 1). Output array of { json: { ... } } for Sheets append. Column set: date, post_url, post_title, views_7d, keyword (or similar so orchestrator can build a "prioritize these" list).

5. **Google Sheets — Append:** documentId and sheetName from config (REFRESH_CANDIDATES_TAB). Option: clear tab before append so only latest run is the list (overwrite), or append and dedupe by post_url — planner choice: overwrite/clear then append so Agent 1 sees only current candidates. If overwrite: use "clear and append" or "update range" if available; otherwise document that workflow appends and orchestrator reads last N rows. Simpler: append only; orchestrator reads all rows and takes unique by post_url. No YOUR_* in JSON.
  </action>

    <automated>test -f growth/06_Refresh_Candidates_Writer.json && grep -q "runReport\|REFRESH_CANDIDATES" growth/06_Refresh_Candidates_Writer.json</automated>
  </verify>
  <done>Refresh Candidates Writer workflow exists; runs on schedule, calls GA4, filters by threshold, writes to REFRESH_CANDIDATES_TAB from config.</done>
</task>
